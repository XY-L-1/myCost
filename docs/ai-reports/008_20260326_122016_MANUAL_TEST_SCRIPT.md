# Manual Test Script

## Purpose

Device/simulator runtime could not be executed in this environment, so this script defines the end-to-end scenarios that should be run manually on iOS and Android.

Run every scenario in:

- English
- Simplified Chinese

If possible, run:

- one guest-only pass
- one authenticated pass
- one guest-to-user merge pass

## Environment Preparation

1. Install dependencies and start the app normally.
2. Use a clean local database for the first pass if possible.
3. Have a working Supabase test account available for sign-in scenarios.
4. If testing merge behavior, start from a fresh guest session with no signed-in user.

## Scenario 1: Guest Flow

### Steps

1. Launch the app with no signed-in session.
2. Confirm the auth entry screen is shown.
3. Tap `Continue without account` / guest entry.
4. Confirm the main tab app opens.
5. Confirm Home, Transactions, Insights, Budget, and Settings tabs are visible.
6. Open Home and verify sync state indicates local-only behavior.

### Expected Results

- App enters guest mode successfully.
- Main navigation is available.
- No auth-only screen appears once guest mode is active.
- Sync messaging does not imply cloud sync is active.

## Scenario 2: Sign-In Flow

### Steps

1. Launch from auth entry.
2. Open Sign In.
3. Enter a valid email/password test account.
4. Submit sign-in.
5. Wait for startup sync/bootstrap to finish.
6. Open Settings and verify sign-out is available.

### Expected Results

- Sign-in succeeds.
- App routes into the main tab shell.
- No startup crash or blank screen occurs.
- Settings shows signed-in state actions.

## Scenario 3: Guest To User Merge

### Steps

1. Enter guest mode.
2. Create:
   - at least 2 expenses
   - 1 custom category
   - 1 budget
   - 1 recurring expense
3. Return to auth flow and sign in.
4. Wait for bootstrap/sync to complete.
5. Inspect Home, Transactions, Budget, Categories, and Recurring screens.

### Expected Results

- Guest-created expenses appear in signed-in history.
- Guest-created category appears in category management and expense editor.
- Guest-created budget remains visible in Budget screen.
- Guest-created recurring rule remains visible in Recurring screen.
- Data should now behave as signed-in scoped data, not guest-only data.

## Scenario 4: Expense CRUD

### Create

1. From Home or Transactions, open the expense editor.
2. Enter amount, description, category, and date.
3. Save.

Expected:

- New expense appears in Home recent transactions.
- New expense appears in Transactions list.
- New expense contributes to month totals in Insights and Budget actuals.

### Edit

1. Open an existing expense from Home or Transactions.
2. Change amount, category, description, and date.
3. Save.

Expected:

- Updated values appear consistently in list/home/insights/budget totals.

### Delete

1. Open an existing expense.
2. Tap delete.
3. Return to history and insights.

Expected:

- Expense disappears from active lists.
- Month/category totals update accordingly.

## Scenario 5: Month Drill-Down

### Steps

1. Open Insights.
2. Create data across at least two different months first if needed.
3. Tap one month.
4. Open `Month detail`.

### Expected Results

- The selected month changes the displayed total.
- Month detail shows category totals only for that month.
- Empty state appears if a month has no category totals.

## Scenario 6: Category Drill-Down

### Steps

1. From Insights, tap a category row for a selected month.
2. Also test the same flow from Month detail.

### Expected Results

- Category transactions screen title matches the category.
- Only transactions from the selected month and selected category appear.
- Dates and amounts render correctly.

## Scenario 7: Language Switching

### Steps

1. Open Settings.
2. Switch to Simplified Chinese.
3. Visit every main tab and the expense editor.
4. Close and relaunch the app.
5. Confirm language persisted.
6. Switch back to English and repeat.

### Expected Results

- Main navigation labels change immediately.
- Settings, auth, home, transactions, insights, budget, and recurring strings change language.
- Startup loading/retry strings use the selected language after relaunch.
- No obvious untranslated fallback strings remain in normal UI flows.

### Watch For

- layout overflow in Chinese labels
- any remaining English fallback strings

## Scenario 8: Budget Flow

### Steps

1. Open Budget.
2. Confirm the local-only note is visible.
3. Set budgets for at least 2 categories in the current month.
4. Add expenses into those categories.
5. Return to Budget.
6. Move month backward and forward.

### Expected Results

- Budget values save and reload correctly.
- Actual values reflect current month expenses.
- Remaining amount changes as expenses change.
- Month navigation updates the data set.

## Scenario 9: Recurring Expense Flow

### Steps

1. Open Settings -> Recurring expenses.
2. Confirm the local-only note is visible.
3. Create:
   - one weekly recurring item
   - one monthly recurring item
4. Toggle one item inactive.
5. Relaunch the app or otherwise trigger startup runtime.
6. Check Transactions and Home recurring count.

### Expected Results

- Recurring rules save and reload.
- Inactive items remain listed but do not generate transactions.
- Due active items materialize into real expenses on startup/runtime.
- Home recurring indicator updates appropriately.

## Scenario 10: Sign-Out / Data Isolation

### Steps

1. While signed in, create identifiable signed-in-only data.
2. Sign out.
3. Confirm app returns to auth flow.
4. Enter guest mode.
5. Inspect Transactions, Insights, Budget, Categories, and Recurring.
6. Sign back into the original account.

### Expected Results

- Guest scope should not show signed-in scoped expenses/categories/budgets/recurring rules.
- Re-signing into the account should restore that user’s scoped data.
- No cross-contamination should occur between guest and signed-in records.

## Scenario 11: Sync Status UX

### Steps

1. In signed-in mode, create or edit an expense/category.
2. Watch Home and Settings sync status.
3. If possible, temporarily disable network and retry.

### Expected Results

- Sync state changes away from idle/local-only in signed-in mode.
- Last sync time updates after successful sync.
- Error state should be visible if sync fails.

## Scenario 12: Category Management

### Steps

1. Open Settings -> Categories.
2. Create a new category.
3. Rename it.
4. Archive it.
5. Restore it.
6. Open the expense editor and confirm category availability changes appropriately.

### Expected Results

- Active and archived sections behave correctly.
- Restored category returns to active use.
- Archived category does not appear as an active choice if the active-only path is used.

## Regression Watchlist

Pay extra attention to:

- guest data appearing in signed-in views
- signed-in data appearing in guest views
- month drill-down showing wrong month/category combinations
- untranslated startup/auth/settings copy
- broken date picker behavior on iOS vs Android
- recurring materialization creating duplicates on repeated launch
- budget totals not matching transaction totals
